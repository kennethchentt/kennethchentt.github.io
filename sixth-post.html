How To Apply Feature Selection

Posted by Kenneth Chen on Thu 08 February 2018

Why, How and When to apply Feature Selection¶

为何要做 Feature Selection ？目的就是为了提取高质量有解释能力的 feature ，最终目的就是在 feature 数量和质量之间取得平衡从而提高模型输出的可靠性。因为过多的数量会导致：

  1. 随着 feature 数量的增加，训练时间指数增长。
  2. 模型会随着 feature 数量增长而出现 overfitting。

好的 Feature Selction 方法可以减少模型输入维度的同时不失去太多信息。除了 PCA 外，以下我们再讨论几个比较受欢迎的方法：

  • Filter Methods
  • Wrapper Methods
  • Embedded Methos

Filter Methods¶

用于计算 features 对模型的重要性 —— feature 与目标变量的关系

1. F-Test¶

F-Test 是一个统计测试，用于模型间的比较，来解释模型间差别是否显著。它的做法是 F-Test 假设两个模型X 和 Y，其中X 是由一个恒量产生，而Y 则是由此恒量和一个 feature 产生。两个模型间的输出差异用 least square error 去判断是否显著。

Scikit learn 提供了 Selecting K best features 来实现

In [ ]:
# for regression task
sklearn.feature_selection.f_regresion

# for classfication task
sklearn.feature_selection.f_classif

但是 F-Test 也有它的弊端，就是它只善于 capture features 和 label 之间的线性关系，它的做法是对于高相关的 feature 给高分，反之低分。然而相关性具有欺骗性，它并不能很好的 capture 强非线性关系。

(Francis Anscombe illustrates how four distinct datasets have same mean, variance and correlation to emphasize ‘summary statistics’ does not completely describe the datasets and can be quite deceptive.)

2. Mutual Information¶

Mutual Information 是用来衡量两个变量的相互独立程度。如果两个变量 X 和 Y：

  1. 如果 X 和 Y 是完全独立，即不能通过 X 推出 Y，反之亦然，则它们的 mutual information 为0.
  2. 如果 X 是 Y 的函数，即通过 Y 可求出 X， 则它们的 mutual information 为1.
  3. 如果 Y=f(X,Z,M,N), 则 0 < mutual information < 1.

Mutual inforamtion 比 F-Test 更好地能 capture 非线性关系。 Sklearn提供以下来实现

In [ ]:
# for regression task
sklearn.feature_selection.mutual_info_regression

# for classfication task
sklearn.feature_selection.mutual_info_classif

F-Test captures the linear relationship well. Mutual Information captures any kind of relationship between two variables.

3. Variance Threshold¶

此方法是当 features 的 variance 低于一定的 cutoff 的时候将其移除。背后idea是当一个 feature 其内部过于相似或者并无太vary，此 feature 一般并不具有多少的预测能力.

sklearn.feature_selection.VarianceThreshold

Wrapper Methods¶

Wrapper Methods 选择 feature 一些子集，以此测量模型的 performance.

1. Foward Search¶

此方法可以帮你逐一以输出模型 performance寻找到最优的 feature，并逐一将其加入 featuer 子集

Forward Selection method when used to select the best 3 features out of 5 features, Feature 3, 2 and 5 as the best subset.

当数据有 n 个 features时：

> 第一轮以每一个feature n 创建n个模型，模型输出预测能力最好的那个模型相对应的feature 被选出加入 feature池
> 第二轮以剩下的feature 创建n-1个模型，模型输出预测能力最好的那个模型相对应的feature 被选出加入 feature池
> 第m轮以剩下的feature 创建n-m个模型，模型输出预测能力最好的那个模型相对应的feature 被选出加入 feature池, 直至 m 个 features被选完为止
