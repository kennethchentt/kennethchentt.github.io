Let's start the training process -- Prepare the features:

  • Training set - for training purpose
  • Validation set - for tuning parameters
  • Test set - out of sample test, to evaluate the performance of the tuned model

We first try to use total N data for Training and validation, and rest data for testing. For feature part, we can use one feature only to start, which is the the Close column, to train the model, and then get the performance data Adjusted R-squared, such that we could compare if extra feature would improve the model performance or not.

(The reason why we use adjusted R-squared instead of R-squared, it's because R-squared will be increased as long as we have a new feature added into the model no matter if the feature is useful or useless, this is due to the its mathematical formula nature. We expect the R-Squared to be larger the better, but with this, it actually doesn't help to evaluate wether the new added feature actually contributes to the model, while adjusted R squared gives a fair battle between numbers of features and its actual perfomance, such that it can identify if a new added feature actually increases the model's metrice or not.)

However, in our case, the fit result is already too good to see the difference, 100% of variances are explained by the model with just using one feature Close, and testing set also yields 98% fitness, so we can't see whether adding extra features really helps to improve the model or not.

Feature elimination¶

Before we dive into the model, we would like to see how importance of all these features thrown into the model are. One may ask, Could we rank them so that we can improve model's efficiency as well as accuracy by deciding which to keep? The answer is yes, and this is very beneficial if we have huge amount of features, although in our case, there's only 4 features, but still good to know how to get the job done.

In our case, if we use 50 data points, we can see features 1(Close),2(MACD) and 4th(ATR) are more significant than features 3rd(stochastic), we can consider to throw away the 3rd feature. But with the data points get bigger, like N=1000, you can see the chart below the Close feature dominates the whole, so we believe the result will be no difference if we use only one feature or all. But it's good to demontstrate the flow of analysis. (Interestingly if you change the target variable to Direction, you would find out the most significant feature becomes ATR, meaning Actual True Range gives better prediction value on predicting market up downs)

(Tips: Sometimes, if dataset is small, we could include it see if the model performance is improved by checking if it increases the metric - Adjusted R-squared, this is explained in later part of this notebook.)

So right now, we will keep all the features for training.

Feature scaling? Side talk about scaling or not¶

For a normal linear regression, scaling the feature values should have no effect (we can try linear model, just uncomment the linear model aboved). The difference here is that sklearn's default SVM kernel isn't linear.

Based on the docs for svm.SVR:

kernel parameter: If none is given, ‘rbf’ will be used.

gamma parameter: Kernel coefficient for ‘rbf’, ‘poly’ and ‘sigmoid’. If gamma is ‘auto’ then 1/n_features will be used instead.

The RBF kernel looks complicated but it's not. It basically just produces 1.0 if a feature vector matches the basis vector, and produces a number that gets closer to 0 as they get more distant. The gamma parameter controls how it declines with distance. The sklearn default gamma parameter is apparently a fixed value based on how many features you have. So, you would expect slightly different results if you scale up and down the feature vectors.

I expect if we change to kernel=‘linear’ we'll get exactly the same predictions.
